# Astra — GitHub Pages + secure worker

Это обновлённая версия бота с более спокойным характером.

## Что изменено
- Astra не обязана комментировать всё подряд.
- У неё есть тактичность (`discretion`) и любопытство (`curiosity`).
- Она может молчать, если кадр/фраза её не зацепили.
- Есть фоновый слух: если к ней не обращаются, она чаще промолчит.
- Есть режимы разговора: обычный, совет, просто выслушай.
- Можно дать ей камеру, экран-шеринг или загрузить скрин.

## Важная граница
Astra **не может сама читать весь экран телефона без твоего разрешения**.
Для этого в интерфейсе есть кнопка **Поделиться экраном**. Если браузер или устройство это не поддерживают, используй кнопку загрузки скрина.

## Структура
- `index.html`
- `app.js`
- `styles.css`
- `worker/index.js`
- `worker/README.md`

## Что грузить на GitHub
Загрузи в корень репозитория:
- `index.html`
- `app.js`
- `styles.css`

Папку `worker/` на GitHub Pages можно тоже хранить, но сам worker запускается отдельно, не в Pages.

## Как это работает
- GitHub Pages = интерфейс
- Cloudflare Worker = backend с `OPENAI_API_KEY`

## Быстрый цикл
1. Залить фронт на GitHub Pages.
2. Создать Cloudflare Worker.
3. Вставить туда `worker/index.js`.
4. Добавить `OPENAI_API_KEY` в secrets worker.
5. В интерфейсе Astra вставить URL worker.


## Новое в версии словарной вариативности
- Добавлен файл `language_library_ru_en.json` с большой RU/EN библиотекой открывающих фраз, эмпатии, советов, вопросов, мягких замечаний и тихих внутренних заметок.
- В `worker/index.js` библиотека встроена прямо в код Worker, так что Astra реально использует вариативные заготовки и помнит недавние формулировки.
- Добавлен анти-повторный слой: Astra избегает недавних открывающих фраз и переупаковывает ответ, если он слишком похож на недавние.
